# Security Policy: Bio-Hazard Protocols

## Supported Versions

| Version | Supported | Notes |
| :--- | :---: | :--- |
| 2.x "Evolved" | ✅ | Current Stable |
| 1.x "Primal" | ❌ | End of Life |

## Reporting a Vulnerability ("Bio-Hazard")

If you discover a security vulnerability (or a "genetic defect" that affects safety), please report it to the **Architects Council** immediately.

**DO NOT create a public GitHub Issue for security vulnerabilities.**

### How to Report
Email **security@krystalstack.org** with the subject "BIO-HAZARD REPORT".
Include:
*   A description of the vulnerability.
*   Steps to reproduce.
*   Potential impact on the "Organism."

### Response Timeline
*   **Acknowledgement**: Within 24 hours.
*   **Assessment**: Within 72 hours.
*   **Patch ("Antibody"):** We aim to release a patch within 14 days of confirmation.

Thank you for keeping the ecosystem safe.
